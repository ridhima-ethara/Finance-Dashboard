Finance Dashboard Overview

Role-based project finance, approvals, provisioning, and actuals in one governed platform

This application connects R&D, TPM, CTO, CFO, and IT workflows across project creation, kickoff, budget building, approvals, key provisioning, daily actuals, change requests, and financial monitoring. The latest implementation also routes project AI access through internal platform tokens instead of exposing provider keys to project teams.

Frontend: React + CRACO + Tailwind-style utility classes Backend: FastAPI + local or Mongo-backed app-state sync Project detail is the primary workspace for budgets, members, tokens, and change requests IT now provisions gateway tokens and logs actuals on the same platform

R&D

Creates projects, runs kickoff, raises testing or sample budgets, logs early-stage task consumption, and raises in-project change requests when scope changes.

TPM

Sees projects once assigned, raises production or general budgets, controls phase execution, logs deliverable progress, and moves approved work into delivery.

CTO / CFO

Review, modify, partially approve, return, or reject budgets. Their decision history is visible back to the requester, and approved requests feed IT provisioning.

IT

Stores provider keys, issues internal platform tokens, applies gateway policy, imports or logs actuals, and feeds real spend visibility back to Finance.

End-to-End Workflow

1

Project setup and kickoff

Projects are created with staged member allocation, project goal, and requirement attachments. Members added during or after kickoff receive the kickoff context.

2

Budget build inside the project

Budgets are raised from the project itself. R&D, testing, sample, rework, production, and general asks all follow the approval chain without separate side modules.

3

Approval and audit visibility

CTO and CFO can edit line items, partially approve, or return requests. Their edits and reasoning are captured in request history so the requester sees what changed.

4

IT provisioning and token issuance

Approved requests move into the IT queue. IT adds the provider key once, defines the gateway policy, and issues internal platform tokens to the assigned project members.

5

Task logging and actuals

Project teams log work in phases or batches, and IT captures actual model, infrastructure, and subscription costs through direct entry or CSV import.

6

Finance monitoring

CFO views approved budgets, actuals, recovery, variance, and governed spend. Deliverable-based claimed-versus-actual analysis can be monitored centrally.

Gateway Token Flow

  1. Member receives an internal platform token from the IT allocation.
  2. Member or app calls the internal AI gateway instead of a model provider directly.
  3. Gateway validates token status, owner, identity, device, network, allowed model, remaining budget, rate limit, and expiration.
  4. Gateway retrieves the provider key securely and keeps it masked from project roles.
  5. Gateway sends the request to the configured provider and model.
  6. Usage and cost are recorded against the mapped employee.
  7. Gateway returns the model response back to the caller.
POST /api/gateway/execute

Key Platform Areas

  • Dashboard: role-specific cards for R&D, TPM, CTO, CFO, and IT.
  • Projects: creation, kickoff, team allocation, budget history, token access, task logging, and change requests.
  • Approvals: clear request detail, modified amounts, decision logs, and routed provisioning.
  • IT dashboard: pending provisioning, gateway token visibility, and daily actuals worksheet.
  • Reports and finance monitoring: portfolio, recovery, warnings, forecasts, and tracked actuals.

To open the live application, go to /login after the frontend server is running.

Persistence

The app syncs browser state with /api/app-state. Local development can persist to a workspace JSON file, while hosted environments can use the backend state store.

Provisioning model

Provider keys and issued platform tokens are tracked per project and per model line, with budget caps, rate limits, network/device policy, and expiry metadata.

IT actuals

Daily actuals accept manual entry or CSV/XLSX import, then roll model, infra, and subscription costs back into the project and finance views.
